## 스킬

- 작업 유형에 맞는 스킬이 `.claude/skills/`에 있으면 우선 활용한다. 스킬은 각자
  description으로 자동 발견되므로 여기에 목록화하지 않는다.
- 스킬 간 우선순위·순서 같은 규칙은 실제로 필요해지면 이 아래에 몇 줄로 추가한다.

## 계획

- 설계 선택이 개입하는 큰 변경은 먼저 계획을 제시하고 승인받은 뒤 구현한다(플랜 모드).
  작고 명확한 작업은 바로 진행한다.

## 커밋

- 하나의 커밋은 하나의 관심사(atomic). 무관한 변경이 섞이면 나눈다.
- 메시지는 한국어 `type(scope): 요약` 형식으로, 무엇을·왜 바꿨는지 드러낸다.
- 프로젝트 검증(테스트·빌드·린트)이 통과한 상태에서 커밋한다.
- 커밋·push는 명시적 요청이 있을 때만 한다. push는 항상 사용자가 판단한다.
