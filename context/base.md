## 작업 원칙

### 코드를 건드리기 전에

- 요구사항과 관련 코드·기존 패턴·해당 테스트를 먼저 이해하고, 어떻게 맞물리는지 파악한 뒤 손댄다.
- 코드의 동작·구조가 불확실하면 추측으로 답하지 않는다 — 해당 파일을 직접 읽어 확인한다.
- 더 단순한 방법이 보이면 말한다. 요청을 넘어서는 기능·추상화·"유연성"은 넣지 않는다.

### 변경은 최소 범위로

- 요청을 해결하는 데 필요한 부분만 바꾼다 — 바뀐 모든 줄이 요청과 직접 연결돼야 한다. 무관한 코드·주석·포맷은 건드리지 않는다.
- 기존 스타일과 컨벤션을 따른다(취향과 달라도).
- 이번 변경이 만든 미사용 요소(import·변수·함수)만 정리한다. 기존 dead code는 언급만 하고 지우지 않는다.

### 검증으로 닫는다

- 작업을 "검증 가능한 목표"로 바꾼다: "동작하게" 대신 "이 입력에서 이 결과가 나오는지 확인".
- 변경이 실제로 동작하는 것을 확인한 뒤에 끝났다고 말한다.
